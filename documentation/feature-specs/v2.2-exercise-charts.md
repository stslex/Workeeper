# Feature spec — v2.2 Per-exercise progress charts (Stage v2.2)

**Status:** Planned. Built on v2.0 foundations (`pagedHistoryByExercise` + `getHistoryByExercise` shipped) and v2.1 PR surfaces (PR card on Exercise detail becomes the in-context chart entry).

This stage ships a full-screen per-exercise progress chart. Architecture choice: **one-shot read on screen load, no `Flow` subscription**. The chart reads `getHistoryByExercise(uuid)` once when the screen opens (or when the user changes selected exercise / preset), buckets and renders client-side. This is the deliberate answer to the v2.1 horizon entry on heavy-aggregation re-execution policy: do not observe heavy aggregations; refresh on user-driven entry instead.

## Architectural notes

### One-shot reads, not `Flow`

Previous stages used Room native `Flow` for everything. v2.2 deviates intentionally for the chart series:

- `getHistoryByExercise` already exists as `suspend` (one-shot).
- The chart screen is a leaf surface. Users open it, look, leave. They are not observing it while another session writes data.
- A `Flow` variant would re-execute the heavy query on every set insert / delete / edit *across the whole DB* (Room invalidates per-table, not per-row). For a chart consumer this is wasted work — by the time data changes, the user is on another screen.
- "Refresh on entry" semantics is enforced by `LaunchedEffect(selectedExerciseUuid, preset)` in the handler. No persistent subscription.

This is the **consumer-side cache pattern** advocated by the v2.1 horizon entry: keep the data in `State`, bound by lifecycle, and let it disappear when the consumer leaves.

### Bucketing semantics

X axis = days. Y axis = a single Y-value per day, computed client-side from raw `HistoryByExerciseRow` rows by:

1. Filter rows to the active preset window (`now - delta..now`, or unbounded for `All`).
2. Group rows by `LocalDate.from(finishedAt, deviceTz)` — device-local day, matches user's calendar perception.
3. Per-day, fold all rows of that day to a single Y-value via the active **metric**:
   - **Heaviest weight (default)**: `max(weight)` over the day's rows. Tooltip lists all reps for the heaviest set. Weightless exercises: `max(reps)`.
   - **Volume per set**: `max(weight × reps)` over the day's rows. Weightless exercises: same as Heaviest weight (volume meaningless without weight); toggle is hidden for weightless.

If two rows tie on the metric, the earlier `finishedAt` wins (consistent with the PR `finished_at ASC` tiebreak). Tooltip then says `"N подходов в этот день"`.

### Edge interpolation

- **Left edge**: when the first point lies after the start of the active window, the line extends flat (`y = firstPointY`) from the left axis to the first point's X.
- **Right edge**: symmetric — flat line at `y = lastPointY` from the last point to the right axis.
- **Between points**: linear interpolation (default Canvas line behavior).
- **Single-point case**: flat line at that single Y across the whole window.
- **Zero-point case**: empty state, no chart canvas drawn.

The "не меняя своего значения" rule applies to both edges: the user reads the X axis as time, and an absence of data on either end should not look like a drop to zero.

## Scope

### 1. New screen — `Screen.ExerciseChart(exerciseUuid: String?)`

Full-screen leaf. Top-bar with title "Прогресс" + back button. Top-right an exercise picker (dropdown / bottom sheet). Below: metric toggle (when exercise is weighted), preset chips (`1М` / `3М` / `1Г` / `Все`), the chart canvas, and a small footer with min/max/last summary.

Entry points (4):

1. **Home top bar icon** (new). Icon left of Settings (per Q5 lock). `exerciseUuid = null` → screen resolves to last-trained exercise via `ExerciseRepository.getLastTrainedExerciseUuid()`.
2. **Exercise detail PR card tap** (v2.1 surface, becomes tappable here). `exerciseUuid` = the card's exercise.
3. **Achievement card tap** — wired in **v2.3** when the Achievement card lands. v2.2 declares the destination only.
4. **Training detail entry** — explicitly deferred per Q1 lock.

When `exerciseUuid = null` and there is no last-trained exercise (fresh install, no finished sessions), the screen shows an onboarding empty state: "Завершите тренировку — здесь появится прогресс" + CTA "Начать тренировку" → navigates to Home.

### 2. Home — chart entry icon

`HomeStore.Action.Click.OnChartsClick`, `HomeStore.Action.Navigation.OpenCharts`. Renders as `IconButton(Icons.Outlined.QueryStats)` left of the existing Settings button in `HomeScreen`'s top bar `actions` slot. `feature_home_charts` string resource.

### 3. Exercise detail — PR card tap

`ExerciseStore.Action.Click.OnPrCardClick` is added; `Action.Navigation.OpenChart(exerciseUuid)` consumed by `NavigationHandler`. The card becomes a `Modifier.clickable { ... }` surface; the v2.1 visual treatment is unchanged.

If `state.personalRecord == null`, no card is shown, so no chart entry from Exercise detail. That is acceptable for v2.2; users without a PR access charts via the Home icon + picker. Follow-up entry below.

## Out of scope

- **Mini chart on Exercise detail** — explicitly dropped per planning. Folded into the full-screen chart entry from PR card.
- **Two points per day** when multiple sessions hit the exercise on one calendar date — collapsed to max-of-day for v2.2. Follow-up below.
- **Make-undone semantics** (mark a set / a performed exercise as "did not happen, ignore in stats") — separate release. Affects every aggregation (PR, volume, history); first non-destructive migration. Tracked separately.
- **Custom date range picker** — presets only for v2.2.
- **Compare-exercises view** (multi-series chart) — not in v2 plan.
- **Export chart as image** — not in v2 plan.
- **Achievement card → chart navigation wiring** — v2.3 deliverable. v2.2 declares `Screen.ExerciseChart` so v2.3 can target it.
- **Standalone Stats dashboard** (multiple charts on one screen, totals, aggregates) — v2.3.

## Locked decisions

- **Q1.** Mini chart on Exercise detail dropped. Full-screen only.
- **Q2.** Per-day bucketing: max-of-day. Two-points-per-day deferred.
- **Q3.** Y metric: toggle between **Heaviest weight** (Y = `weight`, reps in tooltip) and **Volume per set** (Y = `weight × reps`). Default is **Heaviest weight** to match PR semantics. Toggle hidden for weightless (Y = `reps`).
- **Q4.** Default `exerciseUuid = null` resolution: last-trained exercise (`MAX(sn.finished_at)` over finished sessions joined to performed_exercise).
- **Q5.** Standalone entry: `Icons.Outlined.QueryStats` icon left of Settings in Home top bar.
- **Tooltip on point tap.** Exercise name + date + weight × reps (or just reps for weightless) + `"N подходов в этот день"` if `N > 1`. Tap on the tooltip itself navigates to `Screen.PastSession(sessionUuid)` for the session containing the max-of-day set (unambiguous: tied maxes break by earliest `finishedAt`, and that row's `sessionUuid` is the navigation target).
- **Edge interpolation.** Flat fill-forward both edges; linear between points; single-point case = flat across window.
- **Empty state.** Zero finished-session sets for selected exercise → "Нет данных" + CTA "Начать тренировку" (nav to Home).
- **Presets.** `1М` / `3М` / `1Г` / `Все`. Default = `3М`.
- **Render**. Compose `Canvas`, no third-party chart library.
- **Read pattern**. One-shot per screen entry / selection change / preset change. No `Flow` subscription.

## Data layer — DAO + repo extensions

### `ExerciseDao.getLastTrainedExerciseUuid` (new)

```kotlin
@Query(
    """
    SELECT pe.exercise_uuid AS uuid
    FROM performed_exercise_table pe
    JOIN session_table sn ON sn.uuid = pe.session_uuid
    WHERE sn.state = 'FINISHED'
      AND sn.finished_at IS NOT NULL
    ORDER BY sn.finished_at DESC
    LIMIT 1
    """,
)
suspend fun getLastTrainedExerciseUuid(): Uuid?
```

Returns the exercise from the most recent finished session. `null` when no finished sessions exist (fresh install).

### `ExerciseDao.getRecentlyTrainedExercises` (new)

```kotlin
@Query(
    """
    SELECT e.uuid AS uuid,
           e.name AS name,
           e.type AS type,
           MAX(sn.finished_at) AS last_finished_at
    FROM exercise_table e
    JOIN performed_exercise_table pe ON pe.exercise_uuid = e.uuid
    JOIN session_table sn ON sn.uuid = pe.session_uuid
    WHERE sn.state = 'FINISHED'
      AND sn.finished_at IS NOT NULL
      AND e.archived_at IS NULL
    GROUP BY e.uuid
    ORDER BY last_finished_at DESC
    """,
)
suspend fun getRecentlyTrainedExercises(): List<RecentTrainedExerciseRow>
```

Powers the picker. Excludes archived exercises and exercises with zero finished-session activity. Ordering: most-recently trained first — matches the broader recency-sort backlog item (separate issue, but this query already does the right thing for the chart picker).

### `CommonHandler`

```kotlin
internal class CommonHandler @Inject constructor(
    private val interactor: ExerciseChartInteractor,
    private val resourceWrapper: ResourceWrapper,
) : ChartComponent(), Handler<Unit> {

    override fun invoke(action: Unit) = Unit

    fun init(initialExerciseUuid: String?) {
        scope.launch {
            // 1. Load picker list
            val recents = interactor.getRecentlyTrainedExercises().toUi()

            // 2. Resolve target exercise
            val targetUuid = initialExerciseUuid
                ?: interactor.getLastTrainedExerciseUuid()

            val selected = recents.firstOrNull { it.uuid == targetUuid }

            updateStateImmediate { current ->
                current.copy(
                    recentExercises = recents,
                    selectedExercise = selected,
                )
            }

            if (selected != null) {
                loadChartFor(selected.uuid)
            } else {
                updateStateImmediate { it.copy(isLoading = false, isEmpty = true) }
            }
        }
    }

    fun loadChartFor(exerciseUuid: String) {
        scope.launch {
            updateStateImmediate { it.copy(isLoading = true, activeTooltip = null) }
            val state = stateValue
            val rawHistory = interactor.getHistory(exerciseUuid)
            val (points, footer) = ExerciseChartUiMapper.bucketAndFold(
                history = rawHistory,
                preset = state.preset,
                metric = state.metric,
                exerciseType = state.selectedExercise?.type ?: ExerciseTypeUiModel.WEIGHTED,
                resourceWrapper = resourceWrapper,
                now = System.currentTimeMillis(),
            )
            updateStateImmediate {
                it.copy(
                    points = points,
                    footerStats = footer,
                    isEmpty = points.isEmpty(),
                    isLoading = false,
                )
            }
        }
    }
}
```

`init(initialExerciseUuid)` is called from `ExerciseChartGraph` once (`LaunchedEffect(Unit)`). `loadChartFor(uuid)` is reused on picker change, preset change, and metric change.

`stateValue` is the `Store.State` accessor exposed to handlers per existing pattern (verify against current MVI base — if not present, use `state.value` from the StateFlow inside the handler scope).

The mapper computation (`bucketAndFold`) runs on `Default` dispatcher inside the interactor or via `withContext(defaultDispatcher)` wrapper. Detail in mapper section.

### `ClickHandler`

```kotlin
internal class ClickHandler @Inject constructor(
    private val commonHandler: CommonHandler,
) : ChartComponent(), Handler<Action.Click> {

    override fun invoke(action: Action.Click) {
        when (action) {
            is Action.Click.OnPresetSelect -> {
                if (action.preset == stateValue.preset) return
                updateStateImmediate { it.copy(preset = action.preset, activeTooltip = null) }
                stateValue.selectedExercise?.let { commonHandler.loadChartFor(it.uuid) }
                consume(Event.HapticClick(HapticFeedbackType.SegmentTick))
            }
            is Action.Click.OnMetricSelect -> {
                if (action.metric == stateValue.metric) return
                updateStateImmediate { it.copy(metric = action.metric, activeTooltip = null) }
                stateValue.selectedExercise?.let { commonHandler.loadChartFor(it.uuid) }
                consume(Event.HapticClick(HapticFeedbackType.SegmentTick))
            }
            Action.Click.OnPickerOpen -> updateStateImmediate { it.copy(isPickerOpen = true) }
            Action.Click.OnPickerDismiss -> updateStateImmediate { it.copy(isPickerOpen = false) }
            is Action.Click.OnPickerItemSelect -> {
                val item = stateValue.recentExercises.firstOrNull { it.uuid == action.uuid } ?: return
                updateStateImmediate {
                    it.copy(selectedExercise = item, isPickerOpen = false, activeTooltip = null)
                }
                commonHandler.loadChartFor(item.uuid)
            }
            is Action.Click.OnPointTap -> {
                val tooltip = ExerciseChartUiMapper.toTooltip(
                    point = action.point,
                    exercise = stateValue.selectedExercise,
                    metric = stateValue.metric,
                    resourceWrapper = resourceWrapper,
                )
                updateStateImmediate { it.copy(activeTooltip = tooltip) }
                consume(Event.HapticClick(HapticFeedbackType.LongPress))
            }
            Action.Click.OnTooltipDismiss -> updateStateImmediate { it.copy(activeTooltip = null) }
            Action.Click.OnTooltipTap -> {
                val sessionUuid = stateValue.activeTooltip?.sessionUuid ?: return
                consume(Action.Navigation.OpenPastSession(sessionUuid))
            }
            Action.Click.OnEmptyCtaClick -> consume(Action.Navigation.OpenHome)
            Action.Click.OnBack -> consume(Action.Navigation.PopBack)
        }
    }
}
```

### Mapper — `ExerciseChartUiMapper`

The bucketing function is the core piece. Pure, testable, no DB:

```kotlin
internal object ExerciseChartUiMapper {

    data class FoldResult(
        val points: ImmutableList<ChartPointUiModel>,
        val footer: ChartFooterStatsUiModel?,
    )

    fun bucketAndFold(
        history: List<HistoryEntry>,
        preset: ChartPresetUiModel,
        metric: ChartMetricUiModel,
        exerciseType: ExerciseTypeUiModel,
        resourceWrapper: ResourceWrapper,
        now: Long,
    ): FoldResult {
        val zone = ZoneId.systemDefault()
        val windowStart = preset.startMillis(now)  // null for ALL

        // 1. Flatten HistoryEntry → individual sets with day key
        data class FlatSet(
            val day: LocalDate,
            val sessionUuid: String,
            val finishedAt: Long,
            val weight: Double?,
            val reps: Int,
        )

        val flat = history
            .filter { windowStart == null || it.finishedAt >= windowStart }
            .flatMap { entry ->
                entry.sets.map { set ->
                    FlatSet(
                        day = Instant.ofEpochMilli(entry.finishedAt).atZone(zone).toLocalDate(),
                        sessionUuid = entry.sessionUuid,
                        finishedAt = entry.finishedAt,
                        weight = set.weight,
                        reps = set.reps,
                    )
                }
            }

        if (flat.isEmpty()) return FoldResult(persistentListOf(), null)

        // 2. Group by day, fold per metric
        val pointsByDay = flat.groupBy { it.day }.map { (day, dailySets) ->
            val foldComparator = compareByDescending<FlatSet> { f ->
                metricValue(f, metric, exerciseType)
            }.thenBy { it.finishedAt }                  // earliest wins on tie
            val winner = dailySets.sortedWith(foldComparator).first()
            ChartPointUiModel(
                day = day,
                value = metricValue(winner, metric, exerciseType),
                sessionUuid = winner.sessionUuid,
                weight = winner.weight,
                reps = winner.reps,
                setCount = dailySets.size,
            )
        }.sortedBy { it.day }

        return FoldResult(
            points = pointsByDay.toImmutableList(),
            footer = pointsByDay.toFooter(metric, exerciseType, resourceWrapper),
        )
    }

    private fun metricValue(
        set: FlatSet,
        metric: ChartMetricUiModel,
        type: ExerciseTypeUiModel,
    ): Double = when {
        type == ExerciseTypeUiModel.WEIGHTLESS -> set.reps.toDouble()
        metric == ChartMetricUiModel.HEAVIEST_WEIGHT -> set.weight ?: 0.0
        else /* VOLUME_PER_SET */ -> (set.weight ?: 0.0) * set.reps
    }

    fun toTooltip(
        point: ChartPointUiModel,
        exercise: ExercisePickerItemUiModel?,
        metric: ChartMetricUiModel,
        resourceWrapper: ResourceWrapper,
    ): ChartTooltipUiModel = ChartTooltipUiModel(
        sessionUuid = point.sessionUuid,
        exerciseName = exercise?.name.orEmpty(),
        dateLabel = resourceWrapper.formatMediumDate(point.day),
        displayLabel = formatDisplay(point, exercise?.type, metric),
        setCount = point.setCount,
    )
    // ... formatDisplay, toFooter omitted; straightforward formatting
}
```

`HistoryEntry` (v2.0) groups by session; we re-flatten because chart bucketing is by day, not by session, and a day may contain sets from multiple sessions.

`bucketAndFold` is intended to run on `Dispatchers.Default` from the interactor. For 1-year `All` preset over a frequently-trained exercise (~2000 sets), execution is sub-millisecond — a Kotlin loop, no allocation hotpath.

## Compose — chart canvas implementation notes

### `ChartCanvas`

- Single `Canvas(modifier = Modifier.fillMaxWidth().aspectRatio(16f / 9f))`
- Inputs: `points`, `windowStartDay`, `windowEndDay`, axis tokens, `onPointTap: (ChartPointUiModel) -> Unit`
- Math:
  - X mapping: `dayIndex = ChronoUnit.DAYS.between(windowStartDay, point.day); xRatio = dayIndex / totalDays; xPx = xRatio * canvasWidth`
  - Y mapping: `yRatio = (point.value - minY) / (maxY - minY); yPx = canvasHeight - (yRatio * canvasHeight)`. With padding for tick labels.
  - `minY` / `maxY` derived from `points`. If `min == max` (all values equal, single-point case), shift `min -= 1` and `max += 1` to avoid division-by-zero and produce a centered horizontal line.
- Drawing order (back to front): grid → axes → fill-forward edges (left + right flat segments) → series line → points → tap targets (transparent).
- Tap targets: a `Modifier.pointerInput` that maps tap coordinates to the nearest point within a touch radius (e.g. 24dp). Hit the nearest within radius; otherwise no-op.
- No animation in v2.2.

### `ChartTooltipOverlay`

A small Compose card positioned above (or below if near top edge) the tapped point. Anchored by point's `xPx` / `yPx`, computed from `state.activeTooltip` plus the matching point's geometry. Tap on tooltip → `Action.Click.OnTooltipTap`. Tap outside tooltip → `Action.Click.OnTooltipDismiss`.

### `MetricToggle`

Two-segment toggle (`Heaviest weight` / `Volume per set`). Hidden when `state.selectedExercise?.type == WEIGHTLESS`. Implementation: existing `core/ui/kit` segmented control if one exists, otherwise simple `Row` of two `FilterChip`s.

### `PresetChipsRow`

`Row` of four `FilterChip`s for `1М` / `3М` / `1Г` / `Все`. Single-select.

### `ChartFooterStats`

Three small inline labels under the canvas: `Мин: 80 кг` / `Макс: 110 кг` / `Последнее: 105 кг`. Pre-formatted strings from mapper; no further logic in Compose.

### `ChartEmptyState`

Centered icon + text + button. Reuses `core/ui/kit/components/empty/AppEmptyState` if its API supports a CTA button; otherwise compose locally.
